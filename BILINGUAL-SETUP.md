# ROUQY — دليل التحديث ثنائي اللغة (عربي / إنجليزي)

تم تعديل موقعك الحالي ليدعم اللغتين الإنجليزية والعربية مع دعم كامل لاتجاه RTL،
بدون أي تغيير في التصميم أو الأنيميشن (GSAP) الموجود.

---

## الملفات اللي تغيّرت أو انضافت

| الملف | الحالة | الوصف |
|------|--------|-------|
| `src/data/site-data.json` | **استُبدل** | صار فيه مفتاحان `en` و `ar`، كل المحتوى مترجم |
| `src/lib/lang-context.tsx` | **جديد** | نظام اللغة (Context + hook `useLang`) |
| `src/components/lang-switcher.tsx` | **جديد** | زر تبديل اللغة (يظهر في الهيدر) |
| `src/app/layout.tsx` | **عُدّل** | إضافة الخط العربي + تغليف التطبيق بـ `LangProvider` |
| `src/app/page.tsx` | **عُدّل** | كل النصوص تُقرأ من `useLang()` + عكس اتجاه المعرض في RTL |
| `src/app/globals.css` | **عُدّل** | أُضيفت في آخره قواعد RTL والخط العربي وزر اللغة |

> ملاحظة: ما تم استخدام مسارات `/en` و `/ar` لأن صفحتك صفحة واحدة `'use client'`
> فيها أنيميشن GSAP معقّد بالـ pinning؛ تحويلها لمسارات يكسر الأنيميشن.
> استُخدم بدلها تبديل لغة فوري يحفظ اختيار الزائر في المتصفح.

---

## كيف يشتغل

1. اللغة الافتراضية إنجليزي. أول زيارة: لو متصفح الزائر عربي يفتح عربي تلقائياً.
2. زر اللغة في الهيدر (`ع` / `EN`) يبدّل فوراً.
3. الاختيار يُحفظ، فلما يرجع الزائر يفتح بنفس لغته.
4. عند العربي: وسم `<html>` يصير `dir="rtl" lang="ar"`، الخط العربي يشتغل،
   والمعرض الأفقي ينعكس اتجاهه.

## خطوات التشغيل على مشروعك

ما يحتاج تثبيت أي حزمة جديدة — كل شي موجود أصلاً (`next/font` يكفي).

```bash
npm install      # أو bun install
npm run dev
```

افتح المتصفح وجرّب زر اللغة في الهيدر.

## ملاحظات للتحسين لاحقاً

- **روابط السوشال** في `site-data.json` لسا `#` — حدّثها بحساباتكم الحقيقية.
- **أسماء المشاريع** لا تزال عامة (Modern Living…). لو عندك أسماء مشاريع حقيقية
  بدّلها في `items` داخل `en` و `ar`.
- لو تبي تضيف صفحة تفاصيل لكل مشروع لاحقاً، نقدر نبنيها فوق نفس نظام اللغة هذا.
